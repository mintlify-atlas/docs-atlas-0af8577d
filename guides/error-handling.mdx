## Overview

Kontrak Backend uses structured error responses to help you identify and resolve issues quickly. All errors follow a consistent format and include detailed information for debugging.

## Error Response Structure

All API errors return a JSON response with the following structure:

```json
{
  "success": false,
  "message": "Error description",
  "data": {
    // Additional error context (optional)
  },
  "errors": "Error details" // For internal errors
}
```

## HTTP Status Codes

The API uses standard HTTP status codes defined in `src/constants/http.ts`:

| Status Code | Constant | Meaning | When Used |
|-------------|----------|---------|----------|
| 200 | `OK` | Success | Successful requests |
| 201 | `CREATED` | Created | Resource created successfully |
| 400 | `BAD_REQUEST` | Bad Request | Invalid input or validation errors |
| 401 | `UNAUTHORIZED` | Unauthorized | Authentication required |
| 403 | `FORBIDDEN` | Forbidden | Insufficient permissions |
| 404 | `NOT_FOUND` | Not Found | Resource not found |
| 409 | `CONFLICT` | Conflict | Resource conflict |
| 422 | `UNPROCESSABLE_CONTENT` | Unprocessable | Semantic errors |
| 429 | `TOO_MANY_REQUESTS` | Rate Limited | Too many requests |
| 500 | `INTERNAL_SERVER_ERROR` | Server Error | Unexpected server error |

## Error Types

### Application Errors (AppError)

Structured errors with specific status codes and context:

```typescript
export class AppError extends Error {
  public readonly statusCode: number;
  public readonly data: AppErrorContext | undefined;
  
  constructor(
    message: string,
    statusCode: HttpStatusCode,
    data?: AppErrorContext,
  ) {
    super(message);
    this.statusCode = statusCode;
    this.data = data;
  }
}
```

**Example Usage:**
```typescript
throw new AppError(
  'El archivo Excel no contiene datos válidos',
  BAD_REQUEST
);
```

    ```json

    ```
    
    **Debugging Steps:**
    1. Try opening file in Excel
    2. Check file extension (.xlsx)
    3. Look for file corruption indicators
    4. Verify file wasn't truncated during transfer
    
    **Solution:**
    - Re-save file in Excel as .xlsx
    - Remove macros and VBA code
    - Create new file and copy data

    - Unhide any hidden rows
    - Ensure data is in first sheet

### Validation Errors

<Tabs>
  <Tab title="DNI Validation">
    **Common Errors:**
    ```json
    {
      "row": 2,
      "field": "dni",
      "message": "DNI debe tener exactamente 8 dígitos numéricos."
    }
    ```
    
    **Validation Rule:**
    ```typescript
    dni: z.string()
      .regex(/^\d{8}$/, 'DNI debe tener exactamente 8 dígitos numéricos')
    ```
    
    **Valid Examples:**
    - ✅ `12345678`
    - ✅ `87654321`
    
    **Invalid Examples:**
    - ❌ `1234567` (too short)
    - ❌ `123456789` (too long)
    - ❌ `12-345-678` (contains dashes)
    - ❌ `12.345.678` (contains dots)
  </Tab>
  
  <Tab title="Email Validation">
    **Common Errors:**
    ```json
    {
      "row": 3,
      "field": "email",
      "message": "Invalid email"
    }
    ```
    
    **Validation Rule:**
    ```typescript
    email: z.email('El correo es requerido').trim()
    ```
    
    **Valid Examples:**
    - ✅ `user@example.com`
    - ✅ `juan.perez@company.com.pe`
    
    **Invalid Examples:**
    - ❌ `user@` (incomplete)
    - ❌ `@example.com` (missing local part)
    - ❌ `user.example.com` (missing @)
  </Tab>
  
  <Tab title="Salary Validation">
    **Common Errors:**
    ```json
    {
      "row": 5,
      "field": "salary",
      "message": "El sueldo debe ser mayor a 0"
    }
    ```
    
    **Validation Rule:**
    ```typescript
    salary: z.number()
      .refine((value) => value > 0, {
        message: 'El sueldo debe ser mayor a 0',
      })
    ```
    
    **Valid Examples:**
    - ✅ `3000`
    - ✅ `3000.50`
    - ✅ `S/. 3000` (text format)
    
    **Invalid Examples:**
    - ❌ `0`
    - ❌ `-1000`
    - ❌ `N/A`
    - ❌ Empty cell
  </Tab>
  
  <Tab title="Date Validation">
    **Common Errors:**
    ```json
    {
      "row": 4,
      "field": "entryDate",
      "message": "Fecha debe estar en formato DD/MM/YYYY"
    }
    ```
    
    **Validation Rule:**
    ```typescript
    entryDate: z.union([
      z.string().regex(/^\d{2}\/\d{2}\/\d{4}$/, 'Fecha debe estar en formato DD/MM/YYYY'),
      z.date(),
    ])
    ```
    
    **Valid Examples:**
    - ✅ `01/01/2024`
    - ✅ `15/03/1990`
    - ✅ Excel date cell
    
    **Invalid Examples:**
    - ❌ `1/1/2024` (missing leading zeros)
    - ❌ `2024-01-01` (wrong format)
    - ❌ `01-01-2024` (wrong separator)
  </Tab>
</Tabs>

## Error Handler Middleware

The global error handler (`src/middlewares/error-handle.middleware.ts`) catches all errors:

```typescript
export const errorHandler: ErrorRequestHandler = (
  error,
  _req: Request,
  res,
  _next: NextFunction,
) => {
  if (error instanceof AppError) {
    handlerAppError(res, error);
    return;
  }
  
  logger.error(`${error.stack || error.message}`);
  res.status(INTERNAL_SERVER_ERROR).json({
    success: false,
    message: 'Internal Server Error',
    errors: error.message,
  });
};
```

### AppError Handling

```typescript
const handlerAppError = (res: Response, error: AppError) => {
  res.status(error.statusCode).json({
    success: false,
    message: error.message,
    data: error.data || null,
  });
};
```

    ```bash

      -F "excel=@test.xlsx" \
      -v
    ```
    
    Look for:
    - HTTP status code in response headers
    - `success: false` in JSON body
    - `message` field with error description
    - `data` field with additional context
  </Step>
  
  <Step title="Review Server Logs">
    Check server console for detailed error information:
    
    ```bash
    # Look for error logs
    tail -f logs/app.log
    ```

    The logger outputs:
    - Error stack traces
    - Validation details
    - File processing steps
    - CORS blocking messages

  <Step title="Validate Input Data">

## HTTP Client Error Handling Examples

<CodeGroup>
```javascript JavaScript (fetch)
const uploadExcel = async (file) => {
  const formData = new FormData();
  formData.append('excel', file);
  
  try {
    const response = await fetch('http://localhost:3000/api/excel/upload', {
      method: 'POST',
      body: formData,
    });
    
    const data = await response.json();
    
    if (!response.ok || !data.success) {
      // Handle validation errors
      if (data.data?.validationErrors) {
        console.error('Validation errors:', data.data.validationErrors);
        // Display errors to user
      } else {
        console.error('Error:', data.message);
      }
      throw new Error(data.message);
    }
    
    return data.data;
  } catch (error) {
    console.error('Upload failed:', error);
    throw error;
  }
};
```

```python Python (requests)
import requests

def upload_excel(file_path):
    url = 'http://localhost:3000/api/excel/upload'
    
    try:
        with open(file_path, 'rb') as f:
            files = {'excel': f}
            response = requests.post(url, files=files)
        
        data = response.json()
        
        if not data.get('success'):
            # Handle validation errors
            if 'validationErrors' in data.get('data', {}):
                for error in data['data']['validationErrors']:
                    print(f"Row {error['row']}, {error['field']}: {error['message']}")
            else:
                print(f"Error: {data.get('message')}")
            raise Exception(data.get('message'))
        
        return data['data']
    
    except requests.exceptions.RequestException as e:
        print(f"Upload failed: {e}")
        raise
```

```bash cURL
#!/bin/bash

# Upload with error handling
response=$(curl -X POST http://localhost:3000/api/excel/upload \
  -F "excel=@employees.xlsx" \
  -w "\n%{http_code}" \
  -s)

# Extract status code
status_code=$(echo "$response" | tail -n1)
body=$(echo "$response" | sed '$d')

if [ "$status_code" != "200" ]; then
  echo "Error: HTTP $status_code"
  echo "Response: $body"
  exit 1
fi

echo "Success: $body"
```
